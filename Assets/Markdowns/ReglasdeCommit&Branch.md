# COMMITS

### Queda aclarado que se hará uso de un sistema de Versiones Semántico para manejar el proyecto, donde:

# 0\.	0\.	0  
(Mayor, Menor, Parche)

* **Las versiones Mayor: El cambio en esta versión solamente se verá cuando la primera entrega esté lista. Donde el proyecto pasará a una fase entregable.**

* **Versiones Minor: Esta versión cambiará cada vez que se añade una nueva funcionalidad al proyecto o un módulo pero siempre y cuando se mantenga la compatibilidad.**

* **Versiones Parche: De exclusivo uso para cuando se realizan correcciones o cambios ligeros que no cambian la funcionalidad.**

# COMMITS DE DOCUMENTACIÓN

* **Se pide que subir cualquier commit relacionado con la documentación sea publicado con el título (Documentación: "Título de la publicación") junto a la explicación del mismo en la descripción**

* **En este caso se admite que se suban directamente al Main**

# BRANCHES:

* **Para las ramas decidimos utilizar ramas que se separen por tareas.**

* **Por lo cual queda en escrito la **prohibición** de realizar un commit relacionado con el código directo en la rama “main”.**

* **Se acepta realizar ramas temporales para realizar cambios menores o corrección de imprevistos.**
